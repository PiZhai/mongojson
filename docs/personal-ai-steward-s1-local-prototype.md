# 私人智能管家 S1 单机本地管家原型

本文档承接 [S0 边界与安全基线](personal-ai-steward-s0-boundary-baseline.md)，定义私人智能管家在 S1 阶段的单机本地原型目标。S1 的重点是证明“本地常驻、低风险事件记录、手动任务、工作台入口、基础审计”这条闭环可行，而不是进入三端同步、自主执行或高权限系统控制。

## S1 目标与边界

### 阶段目标

S1 要在一台 Windows 设备上建立最小可用闭环：

- 有一个可启动、可停止、可查看状态的本地 Agent。
- 有本地数据存储，用于保存事件、任务、审计日志和基础配置。
- 有当前项目中的 Web 工作台入口。
- 能记录低风险事件和用户手动输入。
- 能创建、查看、更新和取消手动任务。
- 能展示本地审计日志。
- 能删除采集到的本地事件。

### 用户价值

用户第一次能在一个工作台里看到：

- 本地 Agent 是否在线。
- 最近发生了哪些授权范围内的事件。
- 自己创建了哪些任务。
- 系统记录了哪些采集、任务和工具行为。
- 哪些数据可以删除。

### S1 明确不做

- 不做三端同步。
- 不做 macOS/Linux Agent。
- 不做真实高权限命令执行。
- 不做自动执行任务。
- 不做自动删除文件。
- 不做邮件、聊天、日历、云盘、密钥链深度接入。
- 不做子 Agent 调度。
- 不做长期记忆自动写入。
- 不做复杂时间线聚类和语义搜索。

## 原型组成

S1 由四部分组成：

| 模块 | 目的 | S1 交付形态 |
|---|---|---|
| Local Agent | 本机常驻入口和采集/任务执行底座 | Windows 本地进程，支持启动、停止、心跳 |
| Local Store | 保存最小数据 | 本地数据库或当前后端数据库中的 S1 表 |
| Web Workbench | 用户查看和操作入口 | 当前工具项目中新增私人管家工作台页面 |
| Audit Trail | 最小审计闭环 | 记录采集、任务、删除、配置变更 |

## Local Agent

### 定位

Local Agent 是 S1 的本机进程。它不需要一开始就做成正式 Windows Service，但必须具备长期演进到 Service/Task Scheduler 启动的结构。

### S1 能力

- 启动 Agent。
- 停止 Agent。
- 查询 Agent 心跳。
- 查询 Agent 版本。
- 查询 Agent 当前配置摘要。
- 写入低风险事件。
- 写入基础审计日志。

### S1 启动方式

S1 可以先支持以下任一方式：

- 后端进程内置一个 Agent worker。
- 独立本地 Agent 进程，通过 HTTP 或本地 IPC 与后端通信。

如果实现选择后端内置 worker，文档和代码中必须明确它是 S1 原型形态，后续可拆成独立 Agent。

### Agent 状态

Agent 至少需要以下状态：

- stopped：未运行。
- starting：启动中。
- running：运行中。
- degraded：运行中但部分采集器或存储异常。
- stopping：停止中。
- error：启动或运行失败。

### 心跳信息

心跳至少包含：

- agent_id。
- device_name。
- platform。
- status。
- started_at。
- last_heartbeat_at。
- version。
- enabled_collectors。
- last_error。

## 本地数据存储

### S1 最小数据对象

S1 至少保存以下对象：

- AgentStatus：本地 Agent 当前状态。
- Event：低风险事件和手动事件。
- Task：手动任务。
- AuditLog：采集、任务、删除和配置变更日志。
- LocalConfig：本地采集开关和授权目录配置。

### Event 最小字段

- id。
- type。
- title。
- summary。
- source。
- data_level。
- created_at。
- updated_at。
- deleted_at。
- status。
- device_id。
- audit_id。

### Task 最小字段

- id。
- title。
- description。
- status。
- priority。
- due_at。
- source。
- created_at。
- updated_at。
- completed_at。
- canceled_at。
- audit_id。

### AuditLog 最小字段

- id。
- occurred_at。
- actor。
- action。
- target_type。
- target_id。
- source。
- permission_level。
- data_level。
- input_summary。
- output_summary。
- result_status。
- error_summary。

### LocalConfig 最小字段

- id。
- collector_name。
- enabled。
- scope_summary。
- created_at。
- updated_at。
- audit_id。

## 低风险采集源

S1 只允许接入 S0 允许的低风险采集源。

### 允许采集

- 用户手动输入事件。
- 用户手动导入链接。
- 指定目录的文件新增、修改、删除元数据。
- 剪贴板文本摘要，默认不保存疑似敏感字段原文。
- 系统基础状态摘要，例如磁盘空间、网络状态、Agent 状态。

### 默认不采集

- 私密浏览内容。
- 浏览器 Cookie、Token、历史全量记录。
- 密码、密钥、凭据。
- 邮件、聊天、日历正文。
- 健康、财务、法律、身份、位置等高敏原始内容。
- 未授权目录的文件内容。

### 采集器要求

每个采集器都必须具备：

- 开关。
- 采集范围说明。
- 最近采集时间。
- 最近错误。
- 审计记录。
- 删除该采集源历史数据的入口。

## Web 工作台

S1 工作台可以先做成单页或一个独立路由，但需要覆盖以下视图。

### Agent 状态视图

展示：

- Agent 当前状态。
- 最近心跳时间。
- 启用的采集器。
- 最近错误。
- 本地设备名称。

操作：

- 刷新状态。
- 暂停采集。
- 恢复采集。

### 事件列表视图

展示：

- 最近事件。
- 事件类型。
- 来源。
- 创建时间。
- 数据级别。
- 状态。

操作：

- 查看事件详情。
- 删除事件。
- 隐藏事件。
- 从事件创建任务。

### 手动任务视图

展示：

- 任务标题。
- 状态。
- 优先级。
- 截止时间。
- 来源。
- 更新时间。

操作：

- 创建任务。
- 编辑任务。
- 标记完成。
- 取消任务。
- 删除或归档任务。

### 审计日志视图

展示：

- 最近审计记录。
- 操作者。
- 动作。
- 目标对象。
- 权限级别。
- 数据级别。
- 结果状态。
- 错误摘要。

操作：

- 按动作过滤。
- 按目标类型过滤。
- 查看单条审计详情。

### 数据清理视图

展示：

- 事件数量。
- 任务数量。
- 审计日志数量。
- 各采集器数据量。

操作：

- 删除单条事件。
- 按采集器清理历史事件。
- 导出 S1 本地数据摘要。

## API/接口边界

S1 可以采用当前后端 API 或本地 Agent API。无论实现形态如何，行为边界应稳定。

### 必要接口语义

- 获取 Agent 状态。
- 获取采集器配置。
- 更新采集器开关。
- 获取事件列表。
- 创建手动事件。
- 删除事件。
- 获取任务列表。
- 创建任务。
- 更新任务。
- 完成任务。
- 取消任务。
- 获取审计日志。
- 导出本地数据摘要。

### 接口约束

- 删除必须写入 AuditLog。
- 采集器开关变更必须写入 AuditLog。
- 手动任务变更必须写入 AuditLog。
- 默认不提供高权限执行接口。
- 默认不提供跨设备远程调用接口。

## 权限与安全要求

S1 必须继承 S0 基线。

### 允许权限

- A0 无权限：页面展示、说明、空状态。
- A1 只读元数据：Agent 状态、指定目录文件元数据、系统状态摘要。
- A3 低风险本地写入：本应用数据库写入事件、任务、审计日志和配置。

### 不允许权限

- A4 高风险本地写入。
- A5 不可逆操作。
- A6 对外发送。
- A7 系统控制。
- A8 凭据与身份。
- A9 高风险决策。

### 安全规则

- S1 不读取明文凭据。
- S1 不执行 Shell 命令。
- S1 不修改用户文件内容。
- S1 不自动删除用户文件。
- S1 不对外发送用户数据。
- S1 采集器默认关闭或只启用明确安全的手动输入。

## 审计规则

S1 必须记录以下动作：

- Agent 启动。
- Agent 停止。
- 采集器启用或禁用。
- 事件创建。
- 事件隐藏。
- 事件删除。
- 任务创建。
- 任务编辑。
- 任务完成。
- 任务取消。
- 数据导出。
- 采集器错误。

每条审计记录至少满足 S1 AuditLog 最小字段。

## 验收标准

S1 完成时必须满足：

- Agent 可以启动、停止，并能显示 running/stopped/error 等状态。
- Agent 重启后本地事件、任务和审计日志不丢失。
- 用户能在工作台看到最近事件、最近任务和最近审计日志。
- 用户能手动创建任务，并完成或取消任务。
- 用户能删除采集到的任意事件。
- 事件删除、任务变更、采集器变更都会产生审计日志。
- 不存在默认采集敏感目录、私密浏览内容、凭据或高敏原始内容的行为。
- 不存在自动执行命令、自动删除文件、自动发送消息或自动修改系统配置的行为。

## 进入 S2 的门槛

进入 S2 个人数据底座前，必须确认：

- Event、Task、AuditLog 已有稳定 ID。
- Event、Task、AuditLog 已有 created_at、updated_at、deleted_at 或等价字段。
- 事件、任务和审计记录可以被导出。
- 删除行为是可记录的，不是静默移除。
- 工作台已经可以作为后续时间线、意图、记忆和知识页面的入口。
- 低风险采集范围已经清晰可见。
- S1 没有引入 S0 禁止的高风险能力。

## 后续与 S2 的衔接

S2 将在 S1 基础上扩展：

- Event 升级为可转化的统一事件。
- Task 支持从 Event 和 Intent 转化。
- AuditLog 支持更完整的数据变更记录。
- 新增 Intent、Memory、KnowledgeItem、SourceRef、DataTag。
- 引入统一搜索、来源追踪、纠正、导出和软删除策略。
