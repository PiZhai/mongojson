# 私人智能管家需求与目标方向

本文档整理一个三端互通、长期常驻、高权限、可扩展的私人智能管家方向。它不是单纯的聊天机器人，而是一个运行在个人设备与私有数据之上的“个人操作系统级智能层”：能感知、记忆、计划、执行、复盘，并通过当前工具项目提供沟通入口、数据查看入口和可审计的任务控制入口。

## 一句话定位

私人智能管家是一个本地优先、跨 Windows/macOS/Linux 常驻运行的个人 Agent 网络。它负责把用户的日常数据、项目上下文、任务意图、知识资产和自动化工具组织成一个可查询、可执行、可追溯、可反省的个人智能系统。

## 核心目标

1. 长期常驻：在三端稳定后台运行，支持开机自启、断线恢复、离线工作和跨端同步。
2. 高权限可控：能够访问系统、文件、应用、浏览器、命令行和自动化接口，但所有能力必须分级授权、审计和可撤销。
3. 智能感知：能持续收集与用户生活、工作、项目、学习有关的信号，并把低价值噪音过滤掉。
4. 记忆沉淀：把碎片记录、操作历史、偏好、决策、项目事实和知识材料转化成可追溯的记忆库与知识库。
5. 主动执行：支持用户手动创建任务，也能基于观察和记忆主动提出、创建、拆解和执行任务。
6. 多 Agent 协作：具备主 Agent、子 Agent、审计 Agent、工具生成 Agent 等多角色协作能力。
7. 当前项目入口：在本工具项目中提供与管家对话、查看数据、管理任务、审计行为、配置权限和观察系统状态的统一工作台。

## 设计原则

- 本地优先：敏感数据默认留在本机或私有同步节点，不默认上传到第三方平台。
- 用户主权：用户可以查看、导出、删除、纠正、冻结任意记忆、知识、任务和行为日志。
- 高权限不等于失控：高危操作必须经过权限策略、执行计划、影响范围、审计日志和必要时的确认。
- 先可信再自主：自主任务能力必须从建议、草稿、低风险执行开始，逐步扩大到更高权限动作。
- 可解释：重要结论必须能追溯到来源、时间、设备、工具调用和推理依据。
- 可降级：没有模型、网络、同步服务或某台设备离线时，系统仍应保留本地记录、搜索、提醒和手动任务能力。
- 不做监控平台：收集个人数据是为了减轻心智负担，不应变成不可见、不可控的监视系统。

## 总体系统形态

### 设备端守护进程

每台设备运行一个本地常驻 Agent：

- Windows：Windows Service 或 Task Scheduler 启动。
- macOS：LaunchAgent/LaunchDaemon 启动。
- Linux：systemd user service 或 system service 启动。

守护进程负责：

- 采集本机事件和数据。
- 执行本机工具调用。
- 维护本地加密数据库。
- 与其他设备或私有同步节点同步。
- 接收来自 Web 工作台、CLI、快捷键、浏览器扩展、托盘菜单的指令。
- 管理控制面默认只监听本机回环地址；跨设备连接使用独立、最小化并强制设备认证的 Peer 协议面。

### 私有同步层

三端数据互通不应依赖单点云服务。推荐设计：

- 每端都有本地数据库和本地任务队列。
- 私有同步节点可选，用于跨端中继、备份和远程唤醒。
- 数据按类型、敏感级别和保留策略同步。
- 所有同步数据端到端加密。
- 每台设备有独立设备身份、公私钥、信任状态和撤销机制。

### 当前工具项目中的入口

当前项目应提供一个“私人管家工作台”，至少包括：

- 对话入口：与主 Agent 交谈、追问、下达任务。
- 时间线入口：查看跨端日常事件、文件、网页、命令、会议、任务和总结。
- 任务入口：查看手动任务、自主任务、周期任务、子 Agent 执行状态。
- 记忆入口：查看、纠正、合并、删除和冻结记忆。
- 知识入口：搜索文档、网页、笔记、代码、聊天摘要和附件。
- 权限入口：配置每类工具、每台设备、每个子 Agent 的权限。
- 审计入口：查看它做了什么、为什么做、调用了什么工具、影响了哪些数据。
- 仪表盘入口：查看生活、项目、系统、同步、风险和待处理事项。

## D01 高权限能力底座

### 目标

让管家具备足够高的系统操作能力，能完成真实任务，而不是只能给建议。同时必须把权限拆细、记录清楚、允许撤销，避免“全能但不可控”。

### 需求

- 支持文件读写、目录索引、文件移动、归档、删除前预检查。
- 支持 Shell/PowerShell/Bash/Zsh 命令执行。
- 支持应用启动、窗口识别、剪贴板读取和写入、快捷键触发。
- 支持浏览器自动化：读取当前页面、打开链接、填写表单、下载资料、整理标签页。
- 支持代码仓库操作：查看状态、运行测试、生成 diff、提交前检查，但提交/推送必须受策略控制。
- 支持系统信息读取：进程、端口、服务、磁盘、网络、环境变量、启动项。
- 支持外部服务连接：邮件、日历、飞书/Lark、GitHub、云盘、数据库、私有 API。
- 支持凭据调用，但凭据本身不得暴露给模型原文，需通过密钥代理或能力令牌间接使用。

### 权限模型

- 按能力分组：文件、命令、GUI、浏览器、网络、凭据、系统设置、通讯、财务、健康、位置。
- 按风险分级：只读、低风险写入、高风险写入、不可逆操作、对外发送、金钱/法律/医疗相关。
- 按上下文授权：某项目、某目录、某应用、某时间段、某台设备、某类任务。
- 支持审批策略：总是允许、每次确认、只允许草稿、只允许模拟、禁止。
- 支持子 Agent 继承权限上限，子 Agent 不能自行扩大权限。

### 验收标准

- 任意工具调用都能追溯到触发者、任务、设备、时间、参数摘要、结果和影响范围。
- 高风险操作执行前能生成计划、风险和回滚建议。
- 用户能一键暂停所有自主执行，只保留只读观察和提醒。

## D02 智能日常数据收集

### 目标

让系统持续理解用户正在做什么、关心什么、承诺了什么、有哪些重复摩擦，同时避免无差别采集造成隐私负担和存储噪音。

### 可采集信号

- 文件系统：新建、修改、下载、截图、导入、删除、项目目录变化。
- 浏览器：访问历史、标签页、下载、收藏、网页标题、选中文本、稍后读。
- 剪贴板：文本、链接、代码片段、错误日志、订单信息、待办语句。
- 应用活动：当前窗口、活跃应用、工作区切换、会议应用、编辑器项目。
- 开发活动：Git 分支、提交、构建、测试、终端命令、端口、服务状态。
- 沟通活动：邮件、即时消息、会议纪要、日历、承诺事项和等待回复。
- 生活数据：账单、订阅、证件到期、健康提醒、家庭维护、购物清单。
- 系统状态：电量、网络、磁盘、备份、同步状态、服务异常。

### 数据处理要求

- 原始事件和摘要分离保存。
- 默认只保存必要元数据，高敏内容按需保存。
- 支持敏感字段识别、脱敏和本地加密。
- 支持采集黑名单：应用、网站、目录、窗口标题、文件类型。
- 支持“隐私模式”：临时暂停采集，或只记录时间块不记录内容。
- 支持采集解释：每类数据为什么采集、保存多久、用于什么任务。

### 验收标准

- 用户可以查看某天采集了哪些数据源。
- 用户可以对任一事件选择删除、隐藏、标记敏感、转成任务、转成知识或转成记忆。
- 采集系统能区分“噪音事件”和“值得沉淀的事件”。

## D03 记忆库与知识库

### 目标

记忆库负责“关于用户和上下文的长期事实”；知识库负责“外部资料和项目内容”。两者都要可追溯、可更新、可遗忘。

### 记忆库范围

- 用户偏好：沟通风格、技术栈偏好、工具习惯、风险边界。
- 长期目标：学习目标、项目目标、生活目标、健康/财务/家庭目标。
- 项目上下文：仓库结构、运行方式、常见坑、端口、部署习惯、历史决策。
- 人际关系：联系人、偏好、重要日期、承诺事项、沟通边界。
- 工作模式：高效时间段、常用设备、常见任务流、重复摩擦。
- 决策记录：为什么选择某方案、放弃某方案、后续观察点。

### 知识库范围

- 文档、PDF、网页、Markdown、代码片段、会议纪要。
- 邮件、聊天摘要、任务记录、项目说明。
- 文件索引、附件、截图 OCR、链接快照。
- 用户手动加入的资料和 Agent 自动推荐加入的资料。

### 核心要求

- 每条记忆必须有来源、创建时间、置信度、最近验证时间和适用范围。
- 记忆可以被纠正、拆分、合并、降权、归档、删除。
- 知识条目必须保留原始来源或文件位置。
- 支持向量检索、关键词检索、结构化字段过滤和时间过滤。
- 支持“记忆与知识分离检索”：避免把未经确认的资料当成用户事实。
- 支持“只在某项目内生效”的局部记忆。

### 验收标准

- 用户能问“你为什么这么认为”，系统能指出来源。
- 用户能问“你关于我的某项目知道什么”，系统能按事实、偏好、历史决策分组展示。
- 过期或冲突记忆会进入反省队列，而不是永久污染上下文。

## D04 子智能体与工具调用

### 目标

让主 Agent 负责调度和把关，让子 Agent 专注执行具体领域任务。复杂任务由多个 Agent 分工，但最终结果必须统一收敛给用户。

### 子 Agent 类型

- 系统管家 Agent：服务、端口、磁盘、备份、启动项、系统健康。
- 开发 Agent：代码阅读、构建测试、故障诊断、重构建议、提交说明。
- 文档 Agent：资料整理、摘要、知识库归档、Runbook 生成。
- 生活 Agent：家务、账单、证件、旅行、健康记录、购物清单。
- 项目经理 Agent：任务拆解、优先级、阻塞、里程碑、复盘。
- 研究 Agent：资料搜索、对比、引用整理、观点归纳。
- 自动化 Agent：脚本、小工具、快捷命令、工作流生成。
- 审计 Agent：检查权限、风险、证据、遗漏和是否需要用户确认。

### 调度要求

- 主 Agent 维护任务目标、上下文包、权限上限和输出格式。
- 子 Agent 只能访问被授予的上下文和工具。
- 子 Agent 必须输出证据、假设、操作日志和不确定性。
- 长任务支持暂停、恢复、取消、重试和交接。
- 多 Agent 结果必须有冲突合并机制。

### 验收标准

- 一个任务能展示由哪些 Agent 参与、每个 Agent 做了什么、产出了什么。
- 用户能禁用某类子 Agent 或降低其权限。
- 审计 Agent 可以阻止高风险执行。

## D05 手动任务与自主任务

### 目标

既能像任务管理器一样手动创建任务，也能像真正的管家一样主动发现任务、提出建议、创建计划，并在授权范围内执行。

### 手动任务能力

- 快速创建任务：文字、语音、剪贴板、网页、文件、右键菜单。
- 支持一次性任务、周期任务、条件触发任务、项目任务、检查清单。
- 支持任务字段：目标、截止时间、上下文、来源、优先级、风险、验收标准。
- 支持任务分解：大目标拆成可执行步骤。
- 支持任务执行：调用工具、生成文档、整理文件、发送草稿、运行命令。
- 支持任务回顾：完成结果、耗时、失败原因、下次改进。

### 自主任务能力

- 从时间线、剪贴板、邮件、聊天、文件、浏览器和系统事件中发现候选任务。
- 识别承诺、截止日期、未完成事项、重复摩擦、异常事件、长期目标偏差。
- 自主创建任务前先进入“候选任务池”。
- 对低风险任务可自动执行，例如整理下载文件草稿、生成摘要、提醒到期事项。
- 对高风险任务只生成建议、草稿或执行计划。

### 任务类型

- 提醒类：证件到期、账单、订阅、备份、维护。
- 整理类：文件归档、截图 OCR、下载目录清理、链接归类。
- 研究类：资料收集、方案对比、引用整理。
- 开发类：启动项目、诊断报错、更新文档、生成测试建议。
- 沟通类：会议纪要、邮件草稿、等待回复、承诺跟踪。
- 生活类：购物、旅行、家务、健康记录、家庭资产。

### 验收标准

- 用户能清楚区分“我创建的任务”和“系统建议的任务”。
- 自主任务必须保留触发原因。
- 用户可以把某类自主任务设为自动通过、总是询问或永不创建。

## D06 通用 Agent 能力兼容

### 目标

“其他 Agent 能做的一切”不能理解为无边界承诺，而应落地为一个开放的能力运行时：任何外部 Agent 常见能力，都能通过工具、插件、子 Agent、MCP/API 或脚本接入，并纳入同一套权限、记忆、任务和审计体系。

### 能力范围

- 对话问答、长文写作、代码生成、代码审查、测试生成。
- 文件处理、PDF/Word/Excel/PPT 处理、图片 OCR、网页摘要。
- 浏览器自动化、表单填写、网页监控、数据抓取。
- 日历、邮件、即时通讯、会议纪要、任务系统。
- 本地命令执行、脚本编写、环境诊断、服务启动。
- 数据库、API、日志、配置、部署和监控辅助。
- 插件安装、工具发现、能力注册、工作流编排。

### 约束

- 所有能力必须声明输入、输出、权限、风险和审计格式。
- 模型生成的结果默认是草稿，除非任务策略允许自动执行。
- 外部服务能力必须明确认证身份和数据边界。
- 不能绕过本系统的权限层直接调用高危工具。

### 验收标准

- 新工具可以注册为可发现能力。
- 新 Agent 可以声明角色、权限和适用任务类型。
- 能力调用结果能进入任务日志、时间线、记忆库或知识库。

## D07 个人时间线重建

### 目标

自动重建“我今天/这周到底做了什么”，把分散在设备、文件、网页、会议、聊天、代码和任务中的事件组织成可搜索、可回放、可转任务的个人时间线。

### 时间线内容

- 活跃设备、应用、窗口和项目。
- 打开的网页、阅读资料、下载文件和保存链接。
- 编辑过的文件、代码仓库、Git 分支、提交和构建。
- 会议、邮件、聊天摘要、承诺事项和等待回复。
- 创建、完成、推迟和失败的任务。
- 系统事件：服务启动失败、磁盘空间、备份、同步异常。
- 生活事件：账单、购物、健康记录、证件、旅行准备。

### 核心功能

- 日视图、周视图、项目视图、设备视图。
- 自动聚类为“工作段”：例如某项目开发、资料调研、会议、文件整理。
- 支持从时间线恢复上下文：打开相关文件、链接、任务和笔记。
- 支持把事件转成知识、记忆、任务、复盘或忽略。
- 支持隐私过滤：隐藏应用、隐藏窗口、隐藏网站、隐藏时间段。

### 输出形式

- 今日摘要：做了什么、推进了什么、卡在哪里。
- 项目摘要：某项目最近有哪些活动、文件、问题和下一步。
- 异常摘要：长时间未推进、重复失败、频繁切换、遗漏承诺。
- 证据包：用于 Agent 诊断或复盘的事件链。

### 验收标准

- 用户能问“昨天晚上我在处理哪个问题”，系统能基于时间线回答。
- 用户能从一个项目时间线直接恢复相关上下文。
- 用户能删除或隐藏任意时间线片段。

## D08 意图捕捉器

### 目标

捕捉用户还没来得及正式记录的意图：搜索过的东西、复制过的链接、说过的承诺、打开后没处理的文件、会议中提到的行动项、反复出现的摩擦。

### 可捕捉意图

- “我应该做”的承诺：聊天、邮件、会议、笔记中的明确行动。
- “我可能想做”的兴趣：搜索、收藏、复制、下载、反复打开。
- “我忘了做”的遗留：未回复、未提交、未整理、未完成表单。
- “我一直被卡住”的阻塞：重复报错、重复查同一问题、项目停滞。
- “我需要准备”的未来事件：旅行、会议、账单、证件、生日、体检。

### 核心功能

- 候选意图池：不直接打扰用户，先收集成可批量处理的候选项。
- 置信度评分：明确承诺高，模糊兴趣低。
- 触发来源：必须展示来自哪段文本、哪个网页、哪个文件或哪个事件。
- 一键转化：转任务、转提醒、转知识、转稍后读、转忽略。
- 学习反馈：用户经常忽略的意图类型降低优先级。

### 验收标准

- 不把所有剪贴板和浏览历史都变成任务。
- 每个候选意图都能解释“为什么被捕捉”。
- 用户可以配置捕捉范围和打扰频率。

## D09 上下文传送门

### 目标

让用户在 Windows、macOS、Linux 之间切换时，不只是同步文件，而是同步“工作现场”：打开了什么、做到哪一步、下一步是什么、相关资料在哪。

### 典型场景

- Windows 上调试项目，晚上切到 Mac 继续阅读资料和写方案。
- Linux 服务器上运行诊断，回到 Windows 工作台查看结果和下一步。
- 台式机上打开多个网页调研，笔记本上继续写文档。
- 一台设备出问题，另一台设备能接手任务上下文。

### 核心功能

- 工作现场快照：当前项目、分支、编辑文件、终端目录、浏览器标签、任务状态。
- 继续工作按钮：在另一台设备打开相关文件、链接、终端目录和任务卡。
- 设备能力识别：某些任务只能在有权限或有环境的设备执行。
- 上下文打包：给 Agent 提供跨端上下文包，避免重复解释。
- 冲突处理：同一项目在两台设备上状态不同，要提示而不是强行覆盖。

### 验收标准

- 用户能在另一台设备看到“上次在哪台设备做到哪里”。
- 任务可以指定执行设备，也可以由系统推荐最合适设备。
- 传送门不会自动打开敏感页面或执行命令，除非用户授权。

## D10 人生/项目仪表盘

### 目标

仪表盘不追求炫酷展示，而是把生活、工作、项目和系统状态转化为行动信号：今天需要处理什么、什么正在失控、什么长期没有推进、什么值得复盘。

### 仪表盘模块

- 今日必须处理：截止、账单、会议、承诺、异常。
- 项目推进：活跃项目、停滞项目、阻塞、下一步。
- 生活维护：家务、证件、订阅、健康事项、家庭资产。
- 知识积累：最近沉淀的资料、待整理资料、过期知识。
- 记忆状态：待确认记忆、冲突记忆、过期记忆。
- 系统健康：三端在线状态、同步延迟、备份、磁盘、服务异常。
- 风险队列：高权限操作、敏感数据、未审计工具调用。

### 设计要求

- 默认只展示需要行动或确认的事项。
- 支持从总览钻取到时间线、任务、记忆、知识和审计日志。
- 支持生活视图、工作视图、项目视图、系统视图。
- 支持低噪音模式：只保留今天必须处理的 3-5 件事。

### 验收标准

- 仪表盘上的每个提醒都能说明来源和建议动作。
- 用户能把某类提醒静音、降级、延期或转成周期规则。
- 仪表盘不能变成所有数据的堆叠页面。

## D11 记忆反省机制

### 目标

记忆不是越多越好。系统必须定期检查记忆是否准确、过期、冲突、敏感或已经不该继续使用。

### 反省触发

- 时间触发：每周、每月、每季度检查。
- 事件触发：用户纠正、项目切换、工具迁移、路径变化、端口变化。
- 冲突触发：新事实与旧记忆不一致。
- 低置信度触发：来源单一、推断性强、长期未验证。
- 敏感触发：包含凭据、隐私、医疗、财务、身份信息。

### 反省动作

- 请求用户确认。
- 自动降权。
- 标记过期。
- 合并重复记忆。
- 拆分混杂记忆。
- 删除敏感或错误记忆。
- 将事实从全局记忆降为项目局部记忆。

### 反省界面

- 待确认记忆队列。
- 冲突记忆对比。
- “系统现在相信什么”概览。
- 最近自动使用过的记忆列表。
- 一键纠正：以后不要这样记、只在某项目使用、永久删除。

### 验收标准

- 系统不能长期使用明显过期的端口、路径、偏好或项目事实。
- 用户纠正一次后，相关任务和 Agent 上下文应立即受影响。
- 所有自动修改记忆的动作必须记录日志。

## D12 自动生成工具

### 目标

当系统发现用户反复做同一类机械工作时，能够自动建议并生成脚本、小工具、快捷命令或工作流，把一次性操作沉淀成长期能力。

### 触发条件

- 同类命令重复执行。
- 同类文件反复整理。
- 同类网页反复打开和复制。
- 同类文档反复生成。
- 同类报错反复诊断。
- 同类数据格式反复转换。

### 生成物类型

- 本地脚本：PowerShell、Bash、Python、Node。
- 项目内工具页面：表单、解析器、转换器、报告器。
- Agent 工作流：多步工具调用和检查清单。
- 快捷命令：托盘菜单、CLI 命令、热键动作。
- 规则：自动分类、自动提醒、自动归档。
- 模板：文档、任务、会议纪要、排障报告。

### 安全要求

- 自动生成的工具默认进入草稿区。
- 必须展示输入、输出、权限、失败风险和回滚方式。
- 首次执行必须手动确认。
- 执行后要记录效果，判断是否值得保留。
- 能删除、禁用和版本回退。

### 验收标准

- 系统能从重复行为提出“是否生成一个工具”的建议。
- 生成工具后能被能力注册中心发现和调用。
- 工具使用数据能反哺是否继续保留。

## D13 多代理会议

### 目标

复杂问题不应只让一个 Agent 给结论。多代理会议让不同角色围绕同一问题给出观点、证据、风险和反驳，最后由主 Agent 汇总成决策材料。

### 适用场景

- 是否重构某项目。
- 如何规划一个长期产品方向。
- 某次故障的根因复盘。
- 大额购买或方案选择。
- 学习路线规划。
- 工作/生活系统过载时的调整。

### 会议角色

- 主持 Agent：定义问题、范围、输出格式和时间预算。
- 专家 Agent：从技术、产品、生活、财务、研究等角度分析。
- 反方 Agent：寻找反例、风险和隐藏成本。
- 审计 Agent：检查证据、权限、隐私和不确定性。
- 执行 Agent：把结论拆成任务和下一步。

### 输出要求

- 问题定义。
- 已知事实和来源。
- 不确定项。
- 方案列表。
- 各方案收益、成本、风险。
- 分歧点。
- 推荐方案。
- 需要用户拍板的问题。
- 后续任务清单。

### 验收标准

- 用户能看到各 Agent 的独立观点，而不只是最终结论。
- 多代理会议不会自动执行高风险结论，只生成决策材料和候选任务。
- 会议结果可沉淀为知识、记忆、项目决策或任务。

## 数据模型草案

| 实体 | 说明 |
|---|---|
| Device | 三端设备身份、系统类型、在线状态、权限、同步密钥 |
| Event | 原始事件或摘要事件，构成时间线 |
| TimelineSegment | 聚类后的时间块，例如项目开发、会议、资料整理 |
| Task | 手动或自主任务，含目标、来源、状态、验收和执行日志 |
| Intent | 候选意图，尚未正式成为任务 |
| Memory | 关于用户、项目、偏好、决策的长期事实 |
| KnowledgeItem | 文档、网页、文件、会议、代码等知识条目 |
| Agent | 主 Agent 或子 Agent 的角色、权限和配置 |
| Tool | 可调用能力，包括系统工具、脚本、插件、外部 API |
| PermissionGrant | 针对工具、Agent、设备、范围的授权记录 |
| AuditLog | 工具调用、权限变更、任务执行和记忆修改日志 |
| DashboardSignal | 仪表盘展示的行动信号 |
| GeneratedTool | 自动生成的脚本、页面、工作流或模板 |

## 非功能性需求

### 安全

- 本地数据库加密。
- 同步数据端到端加密。
- 凭据通过系统密钥链或专用密钥代理保存。
- 高风险工具调用必须审计。
- 支持紧急停止自主执行。
- 支持设备丢失后的远程撤销。

### 隐私

- 默认最小采集。
- 支持应用、目录、网站、时间段黑名单。
- 支持私密模式。
- 支持数据导出和彻底删除。
- 支持按模块设置保留周期。

### 可靠性

- 守护进程崩溃后自动恢复。
- 任务队列可持久化。
- 同步冲突可见，不静默覆盖。
- 本地离线可用。
- 关键任务有失败重试和失败摘要。

### 可观测性

- 显示每台设备在线状态。
- 显示同步延迟和失败原因。
- 显示任务队列积压。
- 显示最近高风险操作。
- 显示数据采集量和存储占用。

### 可扩展性

- 工具能力注册。
- 子 Agent 注册。
- 插件机制。
- 工作流编排。
- 多模型适配。
- 私有 API 和本地脚本接入。

## 分期路线

### Phase 0：概念验证

目标是证明本地常驻、数据采集、任务和工作台入口可行。

- 单机 Windows 守护进程。
- 本地 SQLite 或 PostgreSQL 存储。
- Web 工作台中的对话入口、任务列表、事件列表。
- 剪贴板、文件目录、浏览器手动导入、系统状态基础采集。
- 手动任务创建和简单执行日志。
- 只读记忆库草案。

### Phase 1：三端基础版

目标是形成真正的三端个人管家底座。

- Windows/macOS/Linux 守护进程。
- 设备身份和本地加密数据库。
- 私有同步节点或点对点同步。
- 时间线基础重建。
- 意图候选池。
- 权限中心和审计日志。
- 记忆库、知识库、任务库基础 UI。

### Phase 2：自主任务与子 Agent

目标是从“记录与问答”进入“主动发现与执行”。

- 子 Agent 注册和调度。
- 工具能力注册。
- 自主任务建议。
- 低风险自动执行。
- 记忆反省队列。
- 项目/生活仪表盘。
- 上下文传送门基础能力。

### Phase 3：高级智能管家

目标是形成可长期依赖的个人智能系统。

- 自动生成工具。
- 多代理会议。
- 复杂工作流编排。
- 跨端现场恢复。
- 高级权限策略。
- 长期趋势分析。
- 个人数字孪生式草稿生成和决策辅助。

## 第一版建议范围

第一版不应追求所有能力同时完成。建议优先做：

1. 本地常驻 Agent。
2. 当前项目 Web 工作台入口。
3. 事件时间线。
4. 手动任务库。
5. 意图候选池。
6. 权限和审计日志。
7. 记忆库基础。
8. 文件/剪贴板/浏览器链接的低风险采集。

第一版明确不做：

- 自动删除用户文件。
- 自动发送邮件/消息。
- 自动付款、下单、投资、医疗建议。
- 默认采集密码、私密窗口、敏感应用内容。
- 完整替代密码管理器、财务软件、医疗系统或团队协作平台。

## 关键风险

| 风险 | 说明 | 应对 |
|---|---|---|
| 权限失控 | 高权限 Agent 可能误操作文件、命令、通讯和凭据 | 能力分级、确认策略、审计、模拟执行、回滚 |
| 数据过采集 | 时间线和意图捕捉容易变成监控 | 最小采集、黑名单、私密模式、保留周期 |
| 记忆污染 | 错误或过期事实长期影响 Agent | 来源、置信度、反省机制、用户纠正 |
| 自主任务打扰 | 系统主动创建太多任务增加负担 | 候选池、低噪音模式、反馈学习 |
| 跨端冲突 | 多设备状态和文件不同步导致误判 | 设备身份、冲突提示、不静默覆盖 |
| 工具生成风险 | 自动生成脚本可能有破坏性 | 草稿区、首次确认、权限沙箱、版本回退 |
| 模型幻觉 | Agent 可能编造结论或错误执行 | 证据约束、审计 Agent、来源引用、执行前计划 |

## 成功标准

短期成功：

- 用户可以在一个工作台里看到“今天发生了什么、有哪些任务、有哪些候选意图、它做了什么”。
- 用户可以纠正记忆，并看到后续 Agent 行为改变。
- 用户可以放心给它只读和低风险执行权限。

中期成功：

- 三端上下文可以互通。
- 它能主动发现有价值任务，但不制造噪音。
- 它能调用子 Agent 完成具体任务，并留下可审计证据。

长期成功：

- 它成为用户生活、工作、学习和项目管理的持续智能层。
- 用户不需要记住大量琐事，也不需要重复解释项目上下文。
- 系统能持续自我整理、自我校验、自我扩展，但始终受用户控制。
