# 私人智能管家 R4.7：模型原生工具调用

R4.7 将普通对话从“模型生成私有计划 JSON”改为标准 function calling Agent 流程。

## 已实现

- Runtime ToolSpec 自动转换为模型 function tools。
- 工具说明包含工作模式，而不只是名称和参数。
- 普通回答直接使用自然语言，不再要求 intent JSON。
- 模型通过 `tool_calls` 自主选择一个或多个工具及参数。
- 提醒/持续任务和显式长期记忆写入分别作为 `steward.create_task`、`steward.save_memory` 一等工具提供。
- 后端只把 tool call 转成不可变执行记录，不接受模型自报的权限和风险。
- 工具输出不依赖模型预测的 `expected_output`，由工具验证器确认。
- 本机和多 Agent 执行完成后，将真实工具结果回传模型生成最终答复。
- 数据策略禁止结果外发时，保留本地治理摘要和固定结果反馈。

## 保持不变的边界

- 每次调用仍经过工具输入校验、Runtime Policy、审批、全局停止和 watchdog。
- A4–A7 继续由独立 Broker 和审批证明控制。
- 路径允许根、网络目标、可执行程序白名单及远程设备权限不能由模型绕过。
- 审计、证据和长期记忆以执行内核事实为准，而不是以模型文本为准。

## 后续增强

- 将主动长期记忆检索注册成一等 function tool；当前相关记忆仍由服务端在每轮模型调用前自动检索并注入。
- 持久化完整 Agent turn，使进程重启后可从任意 tool result 继续多轮调用。
- 支持模型在首批工具结果后继续发起新的工具调用批次，并为每轮设置调用数、耗时和费用预算。
