# 私人智能管家 S0 边界与安全基线

本文档是私人智能管家进入真实后台常驻、系统权限、数据采集、跨端同步、子 Agent 调度之前的边界与安全基线。S0 阶段只定义规则、策略、验收门槛和不可跨越的红线，不实现守护进程、同步、权限系统、真实采集或 Agent 调度。

## S0 目标与适用范围

### 阶段目标

S0 的目标是先把“能做什么、不能做什么、绝不自动做什么、如何审计、如何停止”说清楚，避免后续把高权限私人管家做成不可控的后台系统。

S0 必须产出可被后续阶段引用的基线：

- 产品边界：第一版允许设计的能力、明确不做的能力、远期能力边界。
- 权限分级：后续工具、Agent、设备和任务必须使用的权限分层。
- 数据采集分级：不同数据类型的默认采集、保存、同步和展示策略。
- 高风险操作清单：默认禁止自动执行或必须人工确认的操作。
- 审计基线：后续任何工具调用和数据变更必须留下的审计字段。
- 自主能力边界：自主发现、建议、草稿和真实执行之间的分界。
- S0 完成验收：进入 S1 前必须能回答的问题。

### 适用范围

本基线适用于后续所有私人智能管家相关设计和实现，包括：

- 本地守护进程。
- Web 工作台。
- 任务系统。
- 时间线。
- 意图捕捉。
- 记忆库和知识库。
- 权限中心。
- 审计日志。
- 子 Agent。
- 工具调用。
- 自动生成工具。
- 三端同步。

### S0 不做的事情

S0 阶段不做以下实现：

- 不启动真实后台常驻进程。
- 不接入真实系统权限。
- 不采集真实用户数据。
- 不执行真实系统命令。
- 不接入密钥链、浏览器、邮件、聊天、日历或云盘。
- 不实现跨端同步。
- 不实现子 Agent 调度。
- 不实现自主任务执行。

## 产品边界

### 第一版允许设计的能力

第一版可以设计和后续实现以下能力，但必须遵守本文档的权限、采集、审计和非目标约束：

- 工作台入口：对话、任务、事件、时间线、意图、记忆、知识、权限、审计的统一入口。
- 手动任务：用户显式创建、编辑、完成、取消任务。
- 低风险事件记录：用户明确授权目录、链接、剪贴板摘要、系统状态摘要等。
- 时间线基础：把已记录事件按时间展示和轻量聚合。
- 意图候选池：从授权数据中提出候选任务，但不自动执行。
- 记忆库基础：保存用户确认或可追溯来源的偏好、决策和项目事实。
- 知识库基础：保存用户手动导入或授权导入的文档、网页、笔记和摘要。
- 权限中心：展示和配置工具、数据源、设备、Agent 的权限上限。
- 审计日志：展示系统做过什么、为什么做、使用了什么权限、影响了什么数据。
- 一键暂停：暂停自主建议、自主执行和后台工具调用。

### 第一版明确不做的能力

第一版不得实现以下能力：

- 无人值守高风险自动化。
- 自动删除用户文件。
- 自动发送邮件、聊天消息、短信、电话或对外通知。
- 自动付款、下单、转账、投资、退订、续费或签署协议。
- 自动提交代码、推送代码、发布版本或部署服务。
- 自动修改系统设置、启动项、防火墙、代理、证书、权限或安全策略。
- 自动读取、显示、复制或外发明文凭据。
- 自动接入私密窗口、密码管理器、密钥链、浏览器 Cookie 或会话令牌。
- 自动采集医疗、财务、法律、身份、位置、通讯录等高敏原始内容。
- 自动做医疗、法律、金融、关系、雇佣等高风险决策。
- 公共 SaaS 化、团队协作平台化、企业权限体系。

### 远期能力边界

以下能力可以作为远期目标存在，但进入实现前必须再次经过权限、采集、审计和回退设计：

- 三端长期后台常驻。
- 跨端上下文传送门。
- 子 Agent 自动调度。
- 低风险自主任务执行。
- 自动生成工具。
- 多代理会议。
- 跨端同步和远程设备控制。
- 外部服务深度接入。
- 高权限系统操作。

远期能力不得绕过 S0 基线。任何“以后再加”的能力，只能在权限中心、审计日志、停止机制和回退机制就绪后进入实现。

## 权限分级草案

权限按风险从低到高分级。后续任何工具、子 Agent、设备能力和任务类型，都必须声明所需权限级别。

| 级别 | 名称 | 允许行为 | 默认策略 |
|---|---|---|---|
| A0 | 无权限 | 只展示静态文档、说明、空状态和手动输入框 | 默认允许 |
| A1 | 只读元数据 | 读取文件名、路径、时间、大小、应用名、窗口标题、服务状态等摘要 | 可按数据源授权 |
| A2 | 只读内容 | 读取文件内容、网页正文、剪贴板文本、日志、笔记、邮件正文等内容 | 必须显式授权 |
| A3 | 低风险本地写入 | 写入本应用数据库、创建草稿、添加标签、生成摘要、保存任务 | 用户授权后允许 |
| A4 | 高风险本地写入 | 修改、移动、覆盖、重命名用户文件或项目文件 | 每次确认或模拟 |
| A5 | 不可逆操作 | 删除文件、清空记录、清理目录、销毁密钥、重置配置 | 默认禁止自动执行 |
| A6 | 对外发送 | 发送邮件、消息、HTTP 请求、上传文件、发布内容、推送代码 | 默认禁止自动执行 |
| A7 | 系统控制 | 执行 Shell、安装软件、改系统设置、改权限、改启动项、控制 GUI | 默认禁止自动执行 |
| A8 | 凭据与身份 | 访问密钥链、Token、Cookie、SSH Key、密码管理器、身份认证信息 | 默认禁止读取明文 |
| A9 | 高风险决策 | 财务、医疗、法律、雇佣、身份、关系等决策或建议 | 只允许记录和材料准备 |

### 权限默认规则

- A0 可以默认开放。
- A1 可以按数据源授权，例如“只读指定目录元数据”。
- A2 必须由用户明确选择数据源和用途。
- A3 可以在用户授权后自动执行，但必须可撤销。
- A4 必须先支持模拟执行、影响范围预览和回滚建议。
- A5-A8 默认禁止自动执行，除非后续阶段有单独策略文档和显式确认流程。
- A9 不允许系统替用户做结论，只能做记录、整理、问题清单、材料准备和风险提示。

### 权限继承规则

- 子 Agent 的权限不得高于触发任务的权限上限。
- 工具权限不得高于调用它的 Agent 权限上限。
- 设备权限不得因同步自动扩大。
- 用户临时授权必须有过期时间或撤销入口。
- 高权限工具不能通过低权限任务间接调用。

## 数据采集分级草案

数据按敏感度和默认处理策略分级。后续任何采集器都必须声明采集级别、保存内容、保留周期、是否同步、是否进入记忆。

| 级别 | 数据类型 | 示例 | 默认处理 |
|---|---|---|---|
| D0 | 用户手动输入 | 手动任务、手动笔记、手动导入链接 | 可保存，可检索 |
| D1 | 公开资料 | 公开网页标题、公开文档、公开仓库元数据 | 可保存摘要，保留来源 |
| D2 | 本地元数据 | 文件名、路径、大小、修改时间、应用名、窗口标题、进程名 | 仅在授权范围内保存 |
| D3 | 用户内容 | 文件正文、剪贴板文本、网页正文、笔记、代码片段、日志 | 必须显式授权 |
| D4 | 敏感个人内容 | 聊天、邮件、日历、健康、财务、证件、联系人、位置 | 默认不采集原文 |
| D5 | 凭据与会话 | 密码、Token、Cookie、SSH Key、API Key、密钥链条目 | 禁止采集明文 |
| D6 | 高风险个人数据 | 医疗诊断、法律材料、金融账户、身份证件、家庭隐私 | 只做用户主动导入和本地受控记录 |

### 采集默认规则

- 默认最小采集，只记录必要元数据。
- D0 可保存，但必须支持删除和导出。
- D1 可保存摘要，但必须保留来源链接或文件位置。
- D2 只能在用户授权范围内采集，例如指定目录或指定应用。
- D3 必须明确说明采集目的、保存位置、保留周期和是否进入知识库。
- D4 默认只允许用户主动导入或摘要级记录。
- D5 禁止进入模型上下文、日志、记忆库和知识库。
- D6 不允许自动分析出决策结论，只允许整理材料、生成问题清单和提醒。

### 采集边界

后续采集器必须支持：

- 数据源开关。
- 应用黑名单。
- 目录黑名单。
- 网站黑名单。
- 时间段暂停。
- 私密模式。
- 删除单条事件。
- 删除某数据源历史。
- 导出个人数据。
- 查看“为什么采集这条数据”。

## 高风险操作清单

以下操作在 S0 基线下默认禁止自动执行。后续阶段即使实现，也必须至少经过人工确认、模拟执行、影响范围预览和审计记录。

### 文件与数据

- 删除文件、目录、数据库记录或备份。
- 清空下载目录、桌面、回收站、缓存或日志目录。
- 覆盖用户原始文件。
- 批量移动、重命名或归档文件。
- 修改项目源码、配置、锁文件或迁移文件。
- 对数据库执行写入、删除、迁移或批量更新。

### 系统与环境

- 执行 Shell/PowerShell/Bash/Zsh 命令。
- 安装、卸载或升级软件。
- 修改系统设置、安全策略、防火墙、代理、证书、Hosts、启动项。
- 创建、删除或修改系统服务、计划任务、LaunchAgent、systemd unit。
- 读取或修改系统权限、文件 ACL、管理员权限、sudo 配置。

### 对外通信

- 发送邮件、聊天消息、短信、电话、应用内通知。
- 上传文件到外部服务。
- 发布文章、评论、动态或代码。
- 创建、编辑、关闭 issue、PR、审批、工单。
- 对外 API 写入、Webhook 推送或自动表单提交。

### 凭据与身份

- 读取、展示、复制或外发密码、Token、Cookie、SSH Key、API Key。
- 访问密码管理器、浏览器会话、系统密钥链中的明文数据。
- 自动登录、授权、OAuth 授权确认或权限提升。
- 代表用户签署、确认、付款、下单或授权第三方。

### 高风险决策

- 医疗诊断和用药建议。
- 法律判断和合同签署建议。
- 投资、贷款、保险、税务和付款决策。
- 雇佣、关系、家庭纠纷等重大个人决策。
- 身份证件、账户恢复、数字遗产等不可逆流程。

## 审计基线

后续任何工具调用、数据采集、权限变更、任务执行、记忆写入和知识导入，都必须可审计。

### 审计记录最小字段

每条审计记录至少包含：

- 审计 ID。
- 发生时间。
- 触发来源：用户手动、规则触发、Agent 建议、系统任务、外部事件。
- 操作者：用户、主 Agent、子 Agent、工具、设备。
- 设备 ID 和设备名称。
- 任务 ID 或会话 ID。
- 工具或能力名称。
- 权限级别。
- 数据级别。
- 输入摘要，不保存敏感原文。
- 输出摘要。
- 影响范围。
- 执行模式：草稿、模拟、只读、真实执行。
- 用户确认状态。
- 结果状态：成功、失败、取消、阻止、降级。
- 错误信息摘要。
- 回滚或恢复建议。
- 关联来源引用。

### 审计展示要求

用户必须能在工作台中查看：

- 最近采集了哪些数据。
- 最近创建了哪些任务和候选意图。
- 最近写入、修改、删除了哪些记忆。
- 最近调用了哪些工具。
- 最近有哪些高风险操作被阻止或等待确认。
- 某条结论来自哪些事件、文件、网页、记忆或工具输出。

### 审计不可绕过规则

- 没有审计记录的工具调用不得进入正式实现。
- 审计日志不得保存明文凭据。
- 审计日志不得被普通自主任务自动删除。
- 高风险操作即使失败，也必须记录尝试和阻止原因。
- 子 Agent 和自动生成工具必须遵守同一套审计字段。

## 自主能力边界

S0 允许定义自主能力的边界，但不允许真实自主执行。

### S0 允许的自主能力形态

- 候选意图：系统提出“可能要做的事”，等待用户确认。
- 任务草稿：系统生成任务标题、说明、上下文、验收标准和建议步骤。
- 执行计划：系统描述准备怎么做、需要什么权限、可能影响什么。
- 风险提示：系统指出某个操作需要确认或应该禁止。
- 模拟结果：系统在不改变外部状态的情况下展示预计影响。

### S0 禁止的自主能力形态

- 自动执行命令。
- 自动修改文件。
- 自动发送消息。
- 自动创建外部记录。
- 自动同步设备数据。
- 自动写入长期记忆。
- 自动调用高权限工具。
- 自动安装插件或生成可执行工具并启用。

### 自主能力默认策略

- 默认只建议，不执行。
- 用户确认前不改变系统外部状态。
- 用户确认前不对外发送任何内容。
- 用户确认前不写入长期记忆。
- 用户确认前不把敏感内容发送给外部模型或服务。
- 所有候选意图必须显示触发来源。

## 第一版非目标

第一版私人智能管家不是：

- 后台监控系统。
- 密码管理器。
- 财务自动化系统。
- 医疗诊断系统。
- 法律顾问系统。
- 团队协作平台。
- 企业权限平台。
- 通用云端 SaaS。
- 完整 DevOps 平台。
- 完整数据库客户端。
- 自动化脚本市场。
- 无边界通用 AI Chat。

第一版重点是：

- 明确边界。
- 建立工作台入口。
- 建立任务、事件、意图、记忆、知识、权限、审计的基本概念。
- 允许用户手动输入和确认。
- 为后续真实实现预留安全、可审计、可撤销的结构。

## S0 完成验收

S0 完成时，必须能明确回答以下问题。

### 能做什么

- 第一版允许设计哪些页面、数据结构和用户流程。
- 哪些低风险能力可以进入 S1 原型。
- 哪些数据可以在用户授权后保存。
- 哪些任务只能作为候选、草稿或模拟计划。

### 不能做什么

- 哪些能力不属于第一版。
- 哪些数据默认不采集。
- 哪些工具不能接入真实权限。
- 哪些远期能力必须等权限、审计和回退机制完成后再做。

### 绝不自动做什么

- 不自动删除文件。
- 不自动发送对外消息。
- 不自动付款、下单、投资、退订或签署。
- 不自动提交、推送、部署或发布。
- 不自动读取明文凭据。
- 不自动做医疗、法律、金融等高风险决策。

### 如何查看它做过什么

- 通过审计日志查看采集、任务、工具调用、权限变更、记忆写入和高风险阻止记录。
- 通过来源引用查看某个结论、记忆、任务或建议来自哪里。
- 通过时间线查看事件发生顺序和上下文。

### 如何停止自主能力

- 工作台必须提供一键暂停自主建议和自主执行的入口。
- 暂停后只保留手动输入、手动查看、手动删除和只读展示。
- 高风险工具必须有独立禁用开关。
- 设备、数据源、Agent、工具都必须能被禁用或撤销授权。

## 后续进入 S1 的门槛

进入 S1 单机本地管家原型前，必须满足：

- 本文档已经作为后续设计引用入口。
- 总需求文档和多阶段目标文档已经链接到本文档。
- 第一版允许能力和非目标无冲突。
- 权限分级、数据采集分级、高风险操作清单和审计字段已经覆盖 S1 计划能力。
- S1 不会实现任何本文档禁止的真实高风险操作。
