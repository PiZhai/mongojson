# 对话原生私人智能管家产品需求文档

状态：产品目标基线
适用范围：R4.5 之后的对话、执行、采集、记忆、提醒与跨设备能力
优先级：P0

## 1. 产品愿景

产品不是任务记录系统，也不是要求用户学习工具名、路径格式、数据等级或权限编号的执行控制台。它是一个长期常驻的私人智能管家：用户只需要用自然语言沟通，系统负责理解意图、补齐上下文、调用大模型规划、选择设备和工具、完成真实操作、持续汇报，并把有长期价值的信息沉淀为可追溯的个人记忆。

目标交互是：

> 用户像与真人管家交谈一样表达目标；管家自己理解、调查、计划、执行、验证、记忆，并在真正需要用户决策时才提问。

示例：

- “在桌面创建一个叫动漫的文件夹。”系统应解析当前设备的桌面位置并完成创建，而不是要求绝对路径或工具名。
- “把最近下载的动漫按作品整理一下。”系统应检查下载目录、识别文件、提出或执行整理方案，并返回变更摘要。
- “继续昨天在另一台电脑没做完的工作。”系统应恢复任务、记忆和设备状态，选择可用设备继续执行。
- “我最近是不是经常忘记备份？”系统应基于长期活动与备份记录回答，并可建议或建立提醒和自动化。

## 2. 核心产品原则

### 2.1 对话是唯一必需入口

- 用户不需要先创建任务、选择数据来源、指定 Agent、选择执行器或填写权限等级。
- 每条普通消息都由系统判断属于问答、执行、长期目标、提醒、信息采集、偏好纠正还是上下文控制。
- 执行控制面、审计、权限和高级参数继续存在，但只作为高级管理与故障诊断入口。
- 对话中的“继续、暂停、取消、换到另一台电脑、以后都这样、不要再这样”必须能作用于当前上下文。

### 2.2 大模型是认知与规划中枢

- 真实大模型负责意图理解、缺失信息推断、任务拆解、工具选择、结果判断、异常恢复和自然语言沟通。
- 本地规则只负责确定性的快速路径、安全约束和模型不可用时的降级，不得成为主要的自然语言理解器。
- 模型不能直接绕过执行内核；所有外部影响必须转成结构化计划和工具调用。
- 模型上下文不足时，系统必须主动检索持久化记忆、历史对话、任务、知识和设备状态，而不是只依赖当前聊天窗口。

### 2.3 能力缺口应被解决，而不是转嫁给用户

- 系统能力集合必须可扩展，覆盖本机、浏览器、应用、网络服务和跨设备操作。
- 遇到未知表达时，系统应先解析系统位置、搜索已注册能力、组合已有工具、调查环境或生成受控工具。
- 缺少能力时应明确说明缺少什么，并进入能力安装、工具生成或人工授权流程。
- 不得使用“请提供 `tool:<name>`”“请提供白名单程序”或“请填写绝对路径”作为普通用户的默认交互。
- 用户给出的是目标，系统负责把目标编译为执行细节。

### 2.4 安全机制默认隐形

- 删除对话输入区中的 D0/D1 数据来源选择器，以及其他要求普通用户理解内部分类的控件。
- 对话 API 不再要求客户端为普通消息提交 D0/D1 来源分类。
- 隐私敏感度、权限、风险和数据去向由系统根据内容、来源、目标、设备和动作自动判定。
- 内部可以保留数据敏感度、权限级别和审计标签，但它们是安全实现细节，不是用户完成任务的前置表单。
- 低风险操作默认执行；高风险、不可逆、对外发送、凭据、金钱及重大隐私操作只显示一张简洁确认卡。

## 3. 用户体验要求

### 3.1 对话处理闭环

每条消息必须完成以下闭环：

1. 理解用户真实目标和当前会话指代。
2. 检索相关长期记忆、历史对话、任务、知识、设备和实时环境。
3. 判断是直接回答、单次执行、多 Agent 编排、提醒、持续监控还是主动任务。
4. 自动补齐可可靠推断的信息，例如桌面、下载目录、当前项目、常用设备和惯用应用。
5. 只有在存在多个显著不同结果、信息无法可靠推断或需要高风险授权时才提问。
6. 执行后验证真实结果，而不是只报告命令已发出。
7. 将进度、问题、确认和最终证据返回原对话。
8. 提取值得长期保存的事实、偏好、决策、习惯和未完成事项。

### 3.2 沟通方式

- 简单任务只反馈“正在做”和最终结果，不展示冗长计划。
- 长任务持续给出阶段进度、当前阻塞和预计下一步。
- 失败信息必须说明具体原因、已尝试措施、可自动恢复方案以及是否需要用户介入。
- 系统提问应面向结果，例如“同名文件夹已经存在，要合并还是新建一个？”，而不是面向内部实现。
- 用户可以随时追问“你为什么这样做”“你记住了什么”“信息来自哪里”。

## 4. 执行能力需求

### 4.1 文件与系统

- 解析桌面、下载、文档、图片、音乐、视频、回收站、用户目录、项目目录等系统位置别名。
- 创建、读取、写入、复制、移动、重命名、归档、解压、删除和恢复文件或目录。
- 搜索文件、比较内容、识别格式、批量整理、去重和生成目录结构。
- 查询并管理进程、端口、服务、启动项、环境变量、网络、磁盘、电源和系统更新。
- 启动和控制白名单应用；通过 GUI 自动化处理没有 API 的应用。

### 4.2 浏览器与网络

- 打开和理解网页、搜索公开资料、管理标签页、下载文件、填写表单和监控页面变化。
- 复用用户已登录会话，但对发布、提交、购买、发送等外部影响动作执行风险确认。
- 支持 API、Webhook、MCP、数据库和局域网服务调用。
- 所有网络获取结果保存来源、时间和必要快照，供后续验证与记忆引用。

### 4.3 工作与个人服务

- 支持代码仓库、终端、IDE、构建测试、部署和运行状态诊断。
- 支持邮件、日历、即时通讯、会议、云盘、文档、任务系统和知识库。
- 支持账单、订阅、证件、家庭维护、购物和旅行等个人事务提醒。
- 凭据由独立凭据代理或 Broker 使用，模型和普通执行进程不得读取凭据原文。

### 4.4 跨设备与多 Agent

- 自动选择最合适且在线的设备，也接受“这台电脑”“办公室电脑”“另一台电脑”等自然表达。
- 任务可在设备离线后等待、迁移或在恢复连接后继续。
- 单步骤任务使用单 Run；复杂、并行或跨领域任务自动启用多 Agent 编排。
- 子 Agent 共享目标但只获得完成分工所需的上下文、资源和权限。
- 最终结果统一回到原对话，不要求用户分别查看 worker 或设备日志。

### 4.5 能力获取

- 维护可检索的能力注册表，描述输入、输出、平台、权限、风险、验证与回滚方式。
- 当没有现成能力时，系统可以推荐安装连接器、生成脚本或创建新工具。
- 新生成工具必须先在隔离环境验证，再注册为受控能力；高权限工具必须经过 Broker policy。
- 同一类缺口反复出现时，系统应形成自动化建议，而不是反复询问相同细节。

## 5. 主动信息收集与个人理解

### 5.1 采集目标

系统应在用户授权范围内持续收集能降低心智负担的信息，包括：

- 对话、任务、提醒、决策和用户纠正。
- 文件、下载、截图、剪贴板、浏览器、应用窗口和项目活动。
- 邮件、日历、消息、会议纪要、承诺和等待回复。
- Git、构建、测试、服务状态、系统异常和设备健康。
- 账单、订阅、证件到期、家庭维护和其他用户选择接入的个人信息。

### 5.2 自动整理

- 原始采集事件自动去重、聚合、摘要并建立实体和关系。
- 系统自动识别项目、人物、设备、地点、主题、目标、承诺和时间线。
- 自动发现重复操作、常见失败、工作节奏、偏好、重要联系人和潜在提醒。
- 推断必须保留证据、置信度和适用范围；不能把一次行为直接固化为永久习惯。
- 用户纠正具有最高优先级，并立即影响后续规划和记忆。

### 5.3 主动行为

- 对确定的到期、异常、承诺和风险主动提醒。
- 对可安全自动处理的低风险事项按用户习惯执行并汇报。
- 对不确定或高影响事项给出简洁建议或确认卡。
- 支持免打扰时段、提醒频率、合并摘要、稍后提醒和永久忽略。
- 系统必须学习用户对提醒的采纳、推迟和忽略模式，降低无效打扰。

## 6. 持久化记忆与无限上下文准备

### 6.1 目标

当前大模型上下文窗口只是临时工作区，不是系统记忆。所有有价值的对话、事实、证据、任务状态和用户偏好必须持久化，使任何新会话、模型或设备都能恢复必要上下文，并为未来超长甚至无限上下文模型保留完整、可迁移的数据基础。

### 6.2 记忆分层

| 层级 | 内容 | 主要用途 |
|---|---|---|
| 原始历史 | 完整对话、事件、文件引用、工具调用和结果 | 审计、重建和重新理解 |
| 情节记忆 | 某次任务、一天活动、一次决策或一次问题处理过程 | 恢复过去发生了什么 |
| 语义记忆 | 稳定事实、项目知识、人物关系和长期目标 | 回答与规划 |
| 程序记忆 | 用户偏好的工作方式、重复流程、工具和操作习惯 | 个性化执行 |
| 当前状态 | 活跃任务、等待事项、设备状态、近期关注点 | 连续对话与续作 |
| 证据图谱 | 记忆与来源、时间、实体、冲突和替代关系 | 追溯、纠错和反省 |

### 6.3 写入规则

- 用户明确说“记住”“以后都这样”“不要再这样”时立即形成高优先级记忆。
- 系统可从重复证据生成记忆候选；高置信、低敏感且可撤销的偏好可自动生效，其余等待确认或继续观察。
- 每条推断记忆必须保存来源、时间、置信度、适用范围、最近验证时间和版本。
- 新事实与旧记忆冲突时保留两者及时间关系，不静默覆盖历史。
- 任务执行结果、失败模式和用户反馈应形成可复用的程序记忆。

### 6.4 检索与上下文组装

- 每次模型调用前按当前目标检索，而不是把全部历史机械塞入提示词。
- 组合近期对话、活跃任务、相关语义记忆、程序记忆、知识证据和实时设备状态。
- 检索同时使用关键词、向量、实体关系、时间、项目和设备过滤。
- 上下文包记录选入、排除和压缩过程，支持回答“为什么你会记得/认为这些”。
- 摘要不能替代原始历史；原始数据在保留策略允许时持续保存，以便未来模型重新索引和重新总结。

### 6.5 模型与数据可迁移性

- 记忆使用模型无关的结构化格式和稳定 ID，不绑定单一 embedding 模型或供应商。
- 原始数据、摘要、向量和关系分开存储，允许重新生成摘要和向量。
- 保存生成记忆时使用的模型、提示版本、来源集合和算法版本。
- 支持导出、导入、重建索引、跨设备合并和用户完全删除。

## 7. 自主执行与提醒策略

系统按影响而不是按用户可见的内部编号处理：

- 只读调查和可逆的低风险本地操作可以静默执行。
- 符合已学习偏好且有可靠回滚的重复操作可以自动执行。
- 删除重要数据、修改系统安全设置、对外发布或发送、使用高权限凭据、涉及金钱及其他重大影响的操作必须确认。
- 确认卡只展示用户关心的目标、影响、范围、费用、对外对象和可否撤销，不展示内部权限编号。
- 全局暂停、取消、紧急停止、审计、证据治理和 Broker 隔离继续作为不可移除的底层保障。

“能执行一切”在产品上表示能力体系开放、可以持续扩充并尽量自动完成，而不是允许模型绕过操作系统权限、安全策略或用户意志。

## 8. 非功能性需求

### 8.1 连续性

- 服务、模型、设备或网络重启后恢复对话、任务、租约、进度和未发送提醒。
- 对话消息和执行请求必须幂等，避免重试造成重复操作。
- 长任务必须有心跳、watchdog、断线恢复、补偿和最终状态。

### 8.2 性能

- 普通对话在 1 秒内显示已接收状态。
- 可直接执行的本地低风险任务在 3 秒内进入运行状态。
- 记忆检索不能随历史总量线性退化；冷热分层和索引应支持多年数据。
- 自动采集和索引不得明显影响用户前台应用体验。

### 8.3 隐私与主权

- 用户可以查看系统收集了什么、为什么收集、如何使用和保存多久。
- 支持来源、应用、目录、网站、时间段和设备级别的采集关闭与隐私模式。
- 用户可以纠正、锁定、导出、删除记忆，并要求系统忘记某类信息。
- 敏感内容默认本地加密；外发模型前按用户策略进行最小化、脱敏或本地处理。

### 8.4 可验证性

- 每次执行保存目标、计划、工具、设备、输入摘要、结果、验证、影响范围和证据摘要。
- 系统报告“完成”前必须检查用户可观察的最终状态。
- 自动提醒、习惯和记忆都能追溯到触发证据，并能被用户纠正。

## 9. 产品界面要求

默认界面只保留：

- 对话消息列表。
- 自然语言输入框和发送按钮。
- 必要时出现的一张确认卡或澄清问题。
- 当前任务的简洁进度、暂停和取消入口。
- 最终结果、关键证据和可展开详情。

以下内容从默认对话界面移除：

- D0/D1 或其他数据等级选择器。
- Agent、Run、编排模式和设备的强制手动选择。
- `tool:<name>`、绝对路径和白名单程序提示。
- 普通低风险任务的计划审批步骤。

高级控制面仍提供权限策略、设备、工具注册、详细执行图、证据、审计、记忆管理、采集策略和紧急停止。

## 10. 迁移与实施优先级

### P0：恢复自然对话的基本可信度

1. 删除对话输入区 D0/D1 选择器并简化消息 API。
2. 接入真实大模型 Planner，建立模型调用、工具调用和结果回传闭环。
3. 增加 Windows/macOS/Linux 系统位置解析。
4. 补齐目录创建及常见文件系统操作。
5. 将通用“无法安全编译”改为能力发现、具体缺口说明和自动补救。
6. 建立对话、执行和记忆之间的稳定关联 ID。

### P1：通用本机执行

1. 完整文件、进程、服务、应用、终端和浏览器工具集。
2. 环境调查、能力注册、工具生成与隔离验证。
3. 基于结果的自动验证、重试、回滚和证据回流。
4. 设备与项目别名、用户偏好和重复流程学习。

### P2：主动信息与长期记忆

1. 对话、任务、活动、知识和记忆统一检索。
2. 情节、语义、程序和当前状态记忆分层。
3. 自动习惯推断、提醒、承诺跟踪和低打扰策略。
4. 原始历史保留、摘要重建、embedding 重建和模型迁移工具。

### P3：跨设备自主管家

1. 跨设备上下文恢复、任务迁移和离线续作。
2. 多 Agent 自动编排与资源调度。
3. 邮件、日历、消息、云盘和个人事务的持续协同。
4. 基于长期目标和实时信息的主动计划与执行。

## 11. 核心验收场景

以下场景必须通过真实进程、真实数据库和真实操作系统路径验收：

1. 用户输入“在桌面创建文件夹：动漫”，系统不要求 D0/D1、绝对路径或工具名，直接创建并返回可验证路径。
2. 用户输入“把刚才的文件夹改成动画”，系统正确解析“刚才”并重命名。
3. 用户输入“换到另一台电脑继续”，任务在安全边界内迁移或等待目标设备上线。
4. 用户提出多步骤目标，系统自动选择多 Agent，最终只在原对话汇总结果。
5. 用户要求高影响操作，只出现一张可理解的确认卡；拒绝后不发生外部影响。
6. 服务重启后用户说“继续”，系统恢复未完成任务和必要上下文。
7. 新会话询问数月前项目决策，系统从持久化记忆回答并给出原始来源。
8. 用户纠正偏好后，后续执行立即遵循新偏好，旧记忆保留被替代关系。
9. 系统发现稳定的重复行为后形成习惯候选，经过证据积累后建议自动化或提醒。
10. 用户要求“忘记这件事”，相关记忆、索引和派生数据按策略删除并留下不含原文的删除证明。

## 12. 成功指标

- 90% 以上的常见本机自然语言任务无需用户提供路径、工具名或内部参数。
- 低风险任务从消息发送到进入执行的中位时间小于 3 秒。
- 用户每 100 次任务中因内部实现细节产生的追问少于 5 次。
- 跨会话上下文恢复的关键事实准确率达到 95%，且全部可追溯。
- 主动提醒的采纳、稍后处理或明确有用比例持续提升，无效打扰率持续下降。
- 任何高影响外部操作都能找到授权、执行、验证和证据链。

## 13. 与现有架构的关系

- Runtime V2、R2 执行器、R2.5 控制面、R2.6 watchdog、R3 Broker 和 R4 编排继续作为底层执行与安全基础。
- R4.5 现有对话桥接只是起点，不能作为最终自然语言能力边界。
- 现有活动、实体、关系、习惯、洞察和生命周期数据模型继续作为长期记忆基础，但需要补齐对话原文、情节记忆、程序记忆、版本和上下文组装记录。
- 现有内部数据与权限分级可以保留用于策略计算和审计，但必须从普通对话交互中移除并改为自动判定。
