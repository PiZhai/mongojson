# ADR-0002：R2 使用策略约束的用户态执行层，不让模型直接调用操作系统

- 状态：已接受并实现
- 日期：2026-07-16
- 决策范围：R2 自然语言规划、策略执行、用户态系统工具

## 背景

R1 已提供持久化 Run、审批、重试、取消、验证、证据和恢复，但只有 `runtime.echo`，用户仍需手写结构化计划。R2 的目标是让自然语言任务真正落到本机文件、进程、网络和浏览器，同时不能把模型输出当成可信命令，也不能让 Web/API 主进程获得管理员/root 或凭据权限。

## 决策

### 1. Planner 是计划编译器，不是执行器

`POST /api/steward/runs/plan` 先把自然语言编译成结构化步骤，再走 R1 的规范化、策略、不可变计划哈希和持久化入口。Planner 不能直接调用工具、授予权限、修改审批标志或绕过 Run 状态机。

内置本地规则优先处理明确、单动作的中英文指令。可选 OpenAI-compatible Planner 只作为本地规则不支持时的 fallback，并受数据级别上限约束。Run 保存 `planner`、`planner_version`、原始指令、计划摘要和策略摘要，便于追溯计划来源。

### 2. 工具声明是强制策略输入

每个 RuntimeTool 必须声明：

- 权限级别；
- 风险级别；
- 副作用类型；
- 审批模式；
- 幂等模式；
- 默认超时、取消能力和输入输出 schema。

服务端策略引擎重新计算每一步的决定。调用方和模型只能提高审批要求，不能降低工具声明的审批或权限要求。权限高于 Run ceiling 时拒绝建 Run；任何有副作用、非低风险或 `approval_mode=always` 的工具都绑定当前 `plan_hash` 等待批准。

### 3. R2 只提供有边界的当前用户工具

实现工具如下：

| 工具 | 权限 | 副作用 | 审批 | 幂等语义 |
|---|---|---|---|---|
| `fs.list` | A1 | 无 | 不需要 | inherent |
| `fs.read_text` | A1 | 无 | 不需要 | inherent |
| `fs.create_text` | A2 | 写入 | 每次计划 | keyed，可核对相同内容 |
| `shell.exec` | A3 | 进程 | 每次计划 | non-idempotent |
| `web.fetch_text` | A2 | 网络 | 每次计划 | inherent |
| `browser.open_url` | A2 | 应用启动 | 每次计划 | non-idempotent |

文件工具只能访问规范化后的允许根目录；创建工具只创建新文件，不覆盖已有不同内容。命令工具只直接启动明确白名单中的可执行文件，不经过 `cmd`、PowerShell、`sh` 等解释器，也不继承包含密钥的完整服务环境。浏览器工具默认不注册。网络工具限制重定向和响应大小，默认拒绝 loopback、私网、链路本地等地址，并在连接时重新解析目标，降低 SSRF 和 DNS rebinding 风险。

### 4. 恢复按幂等语义分流

- `inherent`：中断后可生成新 attempt 回队列。
- `keyed`：工具必须能根据目标状态协调；例如相同内容的已存在文件视为已协调，不覆盖不同内容。
- `non_idempotent`：中断时外部结果未知，Run 进入 `blocked`，旧批准被撤销，写入 `run.recovery_blocked`，绝不自动重放。用户显式 `resume` 后仍需重新批准。

R2 不声称能通用判断任意进程或 GUI 动作是否已经生效。未知结果必须暴露给用户，不能用自动重试掩盖。

### 5. R2 与 R3 的信任边界保持分离

R2 进程保持普通当前用户身份，不保存或代理管理员/root 凭据，不提供任意 Shell，不修改系统设置，不删除/覆盖文件，不自动发送外部消息。未来需要这些能力时，必须通过 R3 独立 Privilege Broker、短期能力令牌、单次授权和更强验证接入；不能扩张 R2 主进程权限作为捷径。

### 6. 独立功能开关与渐进启用

`STEWARD_RUNTIME_R2=false` 默认关闭。开启 R2 会隐式启用 R1 内核，但生产配置仍建议显式同时记录两个开关。允许根目录、进程、私网主机和浏览器启动分别配置，未配置即拒绝或不注册，而不是使用宽泛默认值。

## 结果与代价

正面结果：自然语言任务可以真实落盘、读取、执行白名单进程和访问受控网络；计划、审批、策略、调用、验证和证据形成同一条持久化链；进程崩溃不会静默重复非幂等动作。

代价：R2 不是通用桌面自动化，也不能执行任意自然语言多步任务；本地规则只覆盖明确句式，远程 Planner 仍可能返回无效计划并被服务端拒绝；`shell.exec` 的白名单需要运维显式维护；R3 之前无法完成需要提权、凭据或不可逆变更的任务。
