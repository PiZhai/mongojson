# ADR-0009: R4.2 受控 Saga 补偿

- 状态：Accepted
- 日期：2026-07-17
- 决策范围：单机编排失败后的显式、逆序、可审计补偿

## 背景

R4.0/R4.1 能取消尚未完成的兄弟节点，但取消不能撤销已经成功产生的外部副作用。系统需要补偿能力，同时不能假设文件、Shell、浏览器或 Broker capability 都具备事务回滚，也不能让模型在失败后临时生成未经审批的“清理命令”。

## 决策

### 补偿必须预先声明

编排只有显式设置 `failure_policy=compensate` 才启用 Saga。每个正向节点可声明 `compensation_steps`；正向和补偿步骤一起进入不可变编排 plan hash，并在创建时通过相同的工具 allowlist、权限、数据级别、运行时间、尝试次数和总节点预算校验。没有预声明的动作不会在失败后动态生成。

### 补偿也是标准节点

当正向 Runtime 子运行以 `failed` 结束时，调度器先取消并排空其他正向运行，然后为所有“已成功且声明补偿”的节点持久化 `kind=compensation` 的合成节点。每个补偿节点保存 `compensation_of_id`，并像普通节点一样获得：

- 独立 Runtime V2 子运行和计划哈希；
- Ed25519 任务绑定委派；
- 独立 worker 消息、租约和崩溃恢复；
- 审批、证据、watchdog、全局暂停与 Broker 约束。

### 严格逆序

正向 DAG 只允许依赖更早节点，因此位置顺序是拓扑顺序。调度器按成功正向节点位置倒序生成补偿，并用依赖链串联，保证后完成的副作用先补偿。R4.2 不并行执行补偿，避免相互覆盖和顺序不确定。

### 终态

父编排增加以下状态：

- `compensating`：正向执行已失败，正在排空或执行补偿；
- `compensated`：所有声明的补偿均通过后置条件验证；
- `compensation_failed`：至少一个补偿失败，其余未完成补偿被取消。

`compensated` 表示补偿计划成功执行，不等价于外部世界绝对恢复原状。原始失败、补偿节点和全部证据仍永久保留。

### 安全优先

委派篡改、策略撤销、全局急停、deadline 和其他 `blocked` 情况不会自动启动新的补偿动作；这些情况保持 fail-closed。用户显式取消也保持“立即停止”语义，不隐式转为补偿。需要取消时补偿必须由后续明确操作发起。

## 边界

- 不提供数据库意义的原子回滚或 exactly-once 外部副作用；工具仍应使用幂等键或可验证后置条件。
- R4.2 不根据自然语言临时推断补偿，也不允许补偿扩张权限。
- 同一正向节点使用原 Agent 执行补偿；独立补偿角色和人工修复工作流留待后续版本。
- 跨设备 Saga、网络分区仲裁和远程补偿不属于本阶段。

## 验收门槛

- 两个成功节点后发生正向失败时，必须生成两个逆序补偿节点；
- 补偿前必须排空仍活动的正向 Runtime/invocation；
- 补偿可通过独立 worker 消息箱运行并继承全局控制 generation；
- 补偿失败进入独立终态且取消剩余补偿；
- 正向、失败和补偿证据必须进入同一父级 manifest；
- 使用真实 PostgreSQL、真实 Server 和不同 PID 的 worker 完成验收。
